## 标框使命（标 posterBox 前内化 · 首要约束）

**目的：** 从汇总帖的每一张 slide 里，为**每场活动单独**识别并切出它对应的**活动海报**。

这些海报在版面上通常是一块**独立的矩形主视觉**（宣传海报、设计图、竖版配图、现场图等）。你的产出是这块矩形的**精确像素外缘** `posterBox`，使裁切结果像一张完整、干净的活动封面：

- **四边贴准海报边缘** — 不多裁 slide 白底/浅底，不少裁海报上的标题、日期、脚标
- **框内只有本场海报** — 不夹进时间地点说明、黄标题条、票价亮点文字、邻场残影、页眉合集标题
- **比例真实** — `w` 与 `h` 反映该海报真实宽高比，裁出来是完整矩形，不是被拉扁或切残的条块

**精度是硬要求。** 坐标偏几十像素就可能裁进说明文字或切掉海报下缘。拿不准能否框干净时，**本场不写 `posterBox`**（走文字封面），优于交出边缘不准的框。

**没有固定版式。** 必须遵守：

1. **每张 slide 单独读图** — 同一笔记里 `01.webp` 与 `13.webp` 排版可以完全不同；**禁止**假设「这个帖子统一是左文右图/左图右文/上下两场」。
2. **每场活动单独找海报** — 同一张 slide 上两场活动，各自对应哪块矩形、在左在右在不在、尺寸多少，都要**当场重新看**；两场 `w×h` 常常不同。
3. **禁止套坐标** — 不复制上一场、上一页、同页另一场、同城另一笔记的 `x,y,w,h`；不用固定比例或城市模板猜位置。
4. **先认图、再量边** — 先确认「本场对应哪一块矩形图」，再用肉眼定左、上、右、下四条外缘，换算为整数像素。

### 标框完成后必写 `vision-slots.meta.json`

与 `vision-slots.json` 同目录：

```json
{
  "labeledAt": "2026-06-15T12:00:00.000Z",
  "labeledBy": "可选，仅作记录"
}
```

`labeledBy` 可选，仅作审计记录；`labeledAt` 必填。

`extract` 会校验 meta + 拒绝模板坐标（见 `lib/xhs-vision-poster-guard.js`）。

## 你的任务（两件事）

对 `data/scrape-cache/xhs/<城市>/<笔记ID>/images/*.webp` **逐张 slide、逐场活动**标注，写入 `vision-slots.json`：

1. **活动信息**：结构化字段 + **一整段 `intro`（介绍）**
2. **`posterBox`**（可选）：仅当本场有独立矩形海报/主视觉且四边能框干净时，标**精确像素裁切框**

## 一、活动字段怎么填

| 字段 | 必填 | 说明 |
|------|------|------|
| `category` | ✓ | 市集/展览/演唱会等 |
| `name` | ✓ | 活动名称（黄标题条文字） |
| `price` |  | 费用简写：「免费」「xx元起」；仅费用，不写长说明 |
| `time` | ✓ | slide 上的时间原文 |
| `address` | ✓ | slide 上的地点原文 |
| **`intro`** | ✓ | **审核台「介绍」全文，见下** |
| `slide` | ✓ | 如 `01.webp` |
| `posterBox` |  | 见第二节 |

### `intro` 怎么写（重要）

- **把 slide 上本场活动的所有介绍性文字收进来**，用**自然段落**呈现，不要挤成一整段
- **不要**拆成 `ticket`、`highlights` 等子字段；**不要**加「门票：」「亮点：」等标签
- **分段规则**（slide 上能分清时必做）：第一段写入场/报名/预约/费用，第二段写活动特色；两段之间**只换一行**（JSON 里写 `\n`），**不要空行**
- 示例（川美市集，`intro` 字段）：
  `"免费入场，可刷本人身份证入校，也可在「平安川美」官号预约。\n千余件学生原创艺术作品集中亮相，作品涵盖陶艺漆器、书法篆刻、国风手作、原创绘画、潮流文创、非遗巧艺等多元品类。"`

## 二、`posterBox` 怎么标（精准裁切）

### 2.0 工作节奏：动态读图（速度与精度折中）

读几张由当前版式复杂度决定，不固定张数。目标是让上下文足够判断版式，又不把整帖塞进一次视觉任务导致疲劳和套坐标。

| 情况 | 做法 |
|------|------|
| 版式稳定、同类连续页 | 可以连续读多张，标完这一组再继续 |
| 版式突变、信息密集、边界难量 | 单独读当前页，标完再继续 |
| **禁止** | 整帖一气读完再统一写坐标；子代理并行合并坐标；跨 slide 复制完全相同 `x,y,w,h` |

同一期本地宝帖常见规律（**只能作辅助，每页 y 仍要当场看**）：

- 每页 3 场，左栏竖海报 + 右栏说明
- 同页三场的 `x`、`w` 往往接近，**`y` 必须按上/中/下三行分别量**
- 纯文字页（如演唱会列表）、左侧时间表 → skip

读图前若缺 jpg：`node scripts/ensure-slide-review-jpg.js <笔记目录> --slide=XX.webp`（或抓取已双写）。**禁止**批量转全帖 PNG。

### 2.1 问题清单（每场必做，先答比例再写像素）

每场先在心里或草稿里回答（**用占 slide 的百分比，不直接猜像素**）：

| # | 问题 | 产出 |
|---|------|------|
| 1 | 本场主视觉在图的哪一块？（上/中/下 + 左/右） | 区域描述 |
| 2 | 是不是**独立矩形**、能单独当封面？ | 是/否 |
| 3 | 四边大概占整图多少？ | **左%、上%、宽%、高%**（0–100） |
| 4 | 裁进去会不会夹说明字/邻场/页眉？ | 是/否 |

→ 2 为 **否**，或 4 为 **是** → **不写 `posterBox`**（文字封面）。

百分比可用心算，或用脚本**仅做算术**换算（脚本不判断哪块是海报）：

```bash
node scripts/poster-box-percent-to-px.js <笔记目录> --slide=03.webp --left=60 --top=20 --width=35 --height=50
```

将输出的 `x,y,w,h` 写入 `vision-slots.json`。坐标仍相对 **`images/XX.webp`**。

### 2.2 何时 skip（装饰小图 vs 列表海报）

**仍应标 `posterBox`：**

- 本地宝/合集帖**左栏活动海报**（常约 190×280、宽约 17–22%）——虽不大，但是该场**独立宣传图**，应裁切
- 竖版快闪/展览/市集海报缩略图

**应 skip：**

- 纯文字 slide（如演唱会列表无图块）
- 时间表、纯说明区、页眉装饰、分隔线
- 横条 banner（高度很矮）、多宫格拿不准哪格属于本场
- 极小 icon 级装饰（宽约 <12% 且高 <160px）

脚本 `⚠ 尺寸建议 skip` 仅作参考；左栏活动海报即使触发宽度提示，**仍应标框**。

### 2.3 每张 slide 单独分析（禁止帖子级版式假设）

**同一笔记里没有统一版式。** 处理每一张 slide 时，从零开始看这张图怎么排：有几场活动、每场海报在哪一块、是竖矩形还是方矩形、海报与说明文字怎么分界。

推荐顺序（**每张 slide、每场都做一遍**）：

```
打开当前 slide 原图
  → 找到页眉/合集标题（永远不进 posterBox）
  → 对本场活动：在图上定位「只属于这一场」的矩形海报区域
  → 四边贴在海报外缘（底色/边框截止线），排除说明文字、黄条、邻场、slide 大白边
  → 换算 x,y,w,h；心里预览裁切结果是否像完整海报
  → 下一场 / 下一张 slide：重新来，不沿用上一场的坐标或「这张是左文右图」的假设
```

下面只是**偶尔出现的排版现象**（帮助你想「海报可能在哪」），**不是分类菜单，不能用来套模板**：

| 你可能看到的现象 | 标框时注意 |
|------------------|------------|
| 说明文字在一侧、彩色海报在另一侧 | 只框海报矩形，说明区一律排除 |
| 同页上下各一场 | 上下各找一块矩形，分别量，坐标互不相同 |
| 整页几乎是一张大海报 | 框海报外缘，排除页眉与页脚水印 |
| 多宫格 / 日历 / 横条小图 | 只框**本场对应那一格**；横条太矮、框不干净 → skip |
| 只有文字、没有独立图块 | skip |

**常见裁切错误（必须避免）：**

| 错误 | 表现 | 修正 |
|------|------|------|
| 框太宽 | 裁切图出现 ⏰📍🎫 或票价说明 | 减小 `w`，右边缘贴在海报底色截止线 |
| 框太高 | 裁切图顶部带黄标题条或上一场脚标 | 增大 `y`，上边缘贴在海报顶边 |
| 框太窄/太矮 | 海报标题、日期、脚标被切掉 | 增大 `w`/`h` |
| 套模板 | 同页两场或跨 slide 用相同 `x,y,w,h` | 每场单独量 |
| 框进文字区 | 裁切图含大段说明黑字 | 调整 `x`/`w`，左缘贴在海报左缘 |
| 框 slide 白底 | 裁切图大面积空白 | 收紧四边到海报墨水区域 |

### 2.4 标框操作步骤（推荐顺序）

1. 按当前版式复杂度决定这次读几张 `images-jpg/XX.jpg`（坐标仍相对 `images/XX.webp`）
2. 对**每张 slide、每场活动**用问题清单定四边百分比
3. 换算为整数 `x,y,w,h`（心算或 `poster-box-percent-to-px.js`）
4. 同页三场：`y` 按行分别量，不要只改 y 复制上一行
5. 当前这组写完 → 继续下一组 slide

### 2.5 坐标系

相对 **`images/XX.webp` 整图**左上角，单位像素：

```json
"posterBox": { "slide": "01.webp", "x": 648, "y": 292, "w": 396, "h": 428 }
```

### 2.6 何时 skip

- 只有文字、没有独立矩形图块
- **占比太小**（宽 <25% slide 宽，或高 <280px）
- 横条小图（高度很矮，裁出来像 banner）
- 多宫格里拿不准哪一格属于本场
- 框不干净会夹邻场/说明
- 拿不准时：**skip 优于坏裁切**

---

> **海报裁切须在本会话读图标框完成**。**禁止**像素/启发式脚本**判断并写入** `posterBox`；`poster-box-percent-to-px.js` 仅允许做 %→px 算术。

## 四、vision-slots 示例

```json
{
  "01_0": {
    "category": "市集活动",
    "name": "川美青年艺术创意市集",
    "price": "免费",
    "time": "持续至6月28日，周末10:00-19:00",
    "address": "四川美术学院（虎溪校区、黄桷坪校区）",
    "intro": "免费入场，可刷本人身份证入校，也可在「平安川美」官号预约。\n千余件学生原创艺术作品集中亮相，作品涵盖陶艺漆器、书法篆刻、国风手作、原创绘画、潮流文创、非遗巧艺等多元品类。",
    "slide": "01.webp",
    "posterBox": {
      "slide": "01.webp",
      "x": 648,
      "y": 292,
      "w": 396,
      "h": 428
    }
  }
}
```

**不要**再写 `ticket`、`highlights` 字段（旧数据会被脚本合并进 `intro`）。
