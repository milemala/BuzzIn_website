# 小红书海报裁切定稿规则

> **Agent 标 `posterBox` 前必读。** 与 [`xiaohongshu-vision-agent.md`](xiaohongshu-vision-agent.md) 配合使用。  
> 裁切 JS 只执行坐标、不做语义判断；**准不准完全取决于 Agent 是否按本节规则读图标框。**

---

## 核心原则（必须遵守）

1. **每张 slide、每场活动单独分析**  
   版式每周、每账号、每城市都可能不同。先看清「这张图怎么排版」，再标框；**禁止**假设所有笔记都是同一种结构。

2. **`posterBox` = 海报本体外缘的最终裁切区域**  
   四边贴在目标海报的底色/边框截止线，不是 slide 白条、不是左侧说明区、不是邻场残影、不是页眉黄条。

3. **每场坐标独立，禁止套模板**  
   - 禁止把 slide01 的 box 复制到 slide06  
   - 禁止同一页多个活动共用一组 `x,y,w,h`（如全写成 `626,289,410,433`）  
   - 禁止按「上行/下行固定比例」猜位置（如 `y=0.23h` / `y=0.62h`）  
   - 旧版 `poster-box-measured.json`、全图扫描猜框脚本**已废弃**，不要复用

4. **保留海报原始宽高比**  
   `extract` 只做 `sharp.extract`，**不缩放**。若 `w:h` 与真实海报不一致，成品要么切掉内容，要么看起来像「被拉扁/拉长」。  
   同一张 slide 上两场活动的 `w`、`h` **可以且常常应该不同**。

5. **宁可 skip，不可硬裁**  
   框不干净、会带进说明文字或邻场 → 不写 `posterBox`，走文字封面。

## 标框流程（每次抓取 / 重裁都做）

```
读 slide 原图（images/*.webp，必要时转 images-png/*.png）
  → 判断本场 crop / skip
  → 若 crop：在 vision-slots.json 写本场专属 posterBox（px）
  → extract + posters-contact-sheet 总览验收
  → 明显坏图：回原 slide 改 box，最多返工一次
  → 满意后再入库
```

可选：标框后用 `node scripts/preview-poster-boxes.js <笔记目录>` 在 slide 上叠加红框，入库前快速自检。

## 如何分析「海报在哪」

**没有万能版式。** 常见变体举例（仅作参考，以当前 slide 实图为准）：

| 版式类型 | 海报可能在哪 | 注意 |
|----------|--------------|------|
| 左文右图（如西安「周末去哪」某期） | 右侧竖条矩形 | 左侧是 NO.x + 时间地点说明，**不要框左侧** |
| 左图右文 | 左侧竖条 | 不要框右侧说明 |
| 上下两行各一场 | 每行各一块主视觉，可能在左或在右 | 上下两场的 box **分别量** |
| 单活动整页海报 | 占 slide 大部分区域 | 排除页眉合集标题、黄边 |
| 多宫格 / 日历拼盘 | 仅框**本场对应**那一格 | 不要框整张日历 |
| 白底设计海报 | 海报内部可能有大面积白色 | 按外边框/阴影定界，不能 flood-fill 穿到 slide 白底 |

**西安笔记 `6a2670f50000000021015c86` 复盘（勿当通用模板）：**

- 结构：每 slide 两场，**左说明 + 右海报**；上/下各一场  
- 曾犯错误：14 场全用同一组坐标 → 有的裁到左侧文字、有的切掉标题日期、有的比例变方  
- 修正后：每场单独 `posterBox`，例如 `05_0` 约 `398×532`（≈0.75），`07_0` 约 `410×542`（≈0.76），**互不相同**

其他城市、其他账号必须重新读图，不能把西安的 `x≈640` 当标准答案。

## 精度标准（好裁切 vs 坏裁切）

| 必须满足 | 视为坏裁切 |
|----------|------------|
| 海报标题、日期、脚标完整在框内 | 裁掉下缘日期 / 上缘主标题 |
| 四边贴海报外缘，不含 slide 底纹 | 左上露 slide 白边或浅蓝条 |
| 只有本场主视觉，无邻场内容 | 带进 ✅ 闹钟、地图、票价、邻场橙条 |
| 输出比例与海报一致（瘦高就瘦高） | 14 张裁出来几乎一样宽高的「模板感」 |
| 景区照、现场照、展品照也可当封面 | 把说明文字区当海报 |

## posterBox 格式

```json
"posterBox": {
  "slide": "05.webp",
  "x": 647,
  "y": 328,
  "w": 398,
  "h": 532
}
```

- `x,y,w,h`：**整数像素**，相对该 `slide` 原图左上角  
- `slide` 与 `vision-slots` 条目的 `slide` 字段一致  
- 每场活动单独一组；`w/h` 反映该海报真实比例

## 裁切后与入库

- `node scripts/extract-xhs-weekly-events.js <笔记目录>`  
- `node scripts/create-poster-contact-sheet.js <笔记目录>` → 必看 `posters-contact-sheet.png`  
- 门禁见 `lib/xhs-poster-quality-gate.js`（过宽/过高/几乎空白会 drop）；**门禁不能替代准确标框**  
- `snap-poster-box-edges.js` 仅可选修边（语义已对、差几像素）；**不要**用来修框错块、白底海报语义错误

有海报入库后：`event-image-compose.js` 按海报比例合成 4:3 封面（瘦高左图右文 + 标题 CTA），见封面规则文档。

## 交付自检清单

- [ ] 已读**每一张** slide，不是只读第一张就套全场  
- [ ] 每场 `posterBox` 四边对准海报外缘，坐标互不相同（除非两张海报尺寸真的一样）  
- [ ] 总览图里无「说明文字当海报」「邻场残影」「标题日期被切」  
- [ ] 抽查输出 `posters/*.jpg` 的宽高比合理（可用 `preview-poster-boxes.js` 对照原图）  
- [ ] 返工最多一次；仍不满意则 skip 该场或请用户指定继续微调  
- [ ] 满意后再 `import-xhs-events-to-review.js`
